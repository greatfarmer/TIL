# 코르도바(Cordova)

## 코르도바 란?
```
코르도바 홈페이지를 방문하시면 첫 화면이 저렇게 띄워져 있어요.
간단하게 내용을 설명 드리자면 HTML, CSS 및 JS가 포함 된 모바일 앱 그리고 단일 코드 기반의 멀티플 플랫폼을 타겟으로 한 무료이자 오픈소스 프레임워크라고 할수 있겠네요.

코르도바에는 굉장히 다양한 명령 툴, 자바스크립트 프레임워크 그리고 코르도바를 뒷받침 해주는 클라우드 서비스가 존재합니다.
홈페이지 상에는 15가지의 툴에 대한 설명이 나와 있으며 그 중에서도 가장 유명한 것이 어도비 폰갭(Adobe PhoneGap)인데요.
폰갭(PhoneGap)은 아파치 코르도바의 가장 인기있는 배포판으로서 간단한 데스크탑 및 개발자 앱을 사용하여 HTML, CSS 및 자바스크립트(JavaScript)를 몇 분 안에 기기의 앱으로 전환할 수 있습니다.
```

## 특징 1. 크로스 플랫폼 워크플로
```
코르도바에서 지원하고 있는 플랫폼은 위의 이미지에서 보이는 것처럼 Android, iOS, Window Phone 등 8개 입니다.
애초에 자바스크립트 자체가 웹표준을 따르고 있어 다양한 플랫폼 장치의 기능에 접근 할 수 있는 기본 컨테이너에 HTML / 자바스크립트(JavaScript) 앱을 포장(Wrapping)하고 별도로 코드를 수정하는 번거로운 작업 없이도 사용 가능한 앱을 개발할 수 있어요.
이러한 기능은 통합 자바 스크립트 API를 통해 제공되므로 현재 시장에 나와있는 거의 모든 휴대 전화와 태블릿을 타겟으로 한, 그리고 앱 스토어에 게시 가능한 일련의 코드를 쉽게 작성할 수 있습니다.
```

## 특징 2. 플러그인
```
웹 뷰(Web view)기반의 하이브리드 앱은 그 특성상 네이티브 앱 보다 다양한 기능을 구현하기 어려운데요.
코르도바에서는 이런 단점 극복을 위한 다양한 플러그인을 제공해 준답니다.
그리고 그 수는 천 여개가 훨씬 넘어갈 만큼 많습니다.
게다가 혹시 필요한 기능의 플러그인이 없다 하더라도 개발자가 직접 플러그인을 개발하는 것 역시 가능합니다.
코르도바에서 제공하는 플러그인들과 플러그인 개발가이드를 통한 새로운 플로그인으로 인해 하이브리드 앱 개발의 단점을 최소로 줄여줍니다.
덕분에 다양한 플랫폼에서의 다양한 요구사항을 맞춘 개발로 One source Multi device를 실현할 수 있습니다.
```

## 특징 3. 다양한 명령 툴
```
앞서 말씀 드린것처럼 코르도바에는 굉장히 다양한 명령 툴들이 있는데요.
유명한 프로젝트 몇 가지들을 소개해 드리려고 합니다.

● Adobe PhoneGap
   : 대표적인 모바일 개발 프레임워크로서 HTML5, CSS, JavaScript를 활용해 모바일 기기를 위한 애플리케이션을 제작할 수 있습니다.

● Ionic
   : Ionic 프레임워크는 앵귤러(Angular JS)와 조화를 이루며 작동합니다.
   또한 다양한 유틸리티를 지원하며 iOS, Android와 같이 각 OS에 따른 개별적 수정도 가능하고 UI 컴포넌트를 제공하며 넓은 사용자 층이 있어 개발자 커뮤니티 또한 굉장히 활발합니다.

● Onsen UI
   : Ionic의 경쟁자인 OnsenUI는 앵귤러(Angular JS)를 지원하며 UI 개발에서 어려움을 겪은 코르도바(Cordova)/폰갭(PhoneGap) 개발자들을 위해 만들어 졌습니다.
   크로스 플랫폼, 코르도바와 폰갭에 친화적인 구성으로 다양한 유틸리티를 제공하고 앵귤러와의 호환성 역시 장점으로 꼽힙니다.
   경쟁자인 Ionic과 비교하여 큰 차이가 있다면 Ionic 커뮤니티가 더 활발하기 때문에 더 좋은 프레임워크를 만들기 위한 노력 역시 더 활발하다는 점입니다.
   또 프레임워크 자체에서도 Ionic이 좀 더 다양한 요소를 제공해 주고 있습니다.

● Visual studio
   : MS사에서 만든 통합 개발 환경툴로서 Visual C/C++와 Visual Basic를 통합하여 발표한 것을 시작으로 많은 개발자들에게 인기가 있습니다.
   Android, iOS 및 Windows 용 크로스 플랫폼 앱을 제작하는 데 굉장한 인기가 있고 고급 빌드 및 디버깅 지원으로 완성됩니다.
```

## 코르도바(Cordova), 장점을 말하다.
```
장점이 매우 많은 코르도바는 하이브리드 앱을 위한 프레임워크이지만 네이티브 앱의 기능을 넣을 수 있어요.
물론 네이티브 앱에 비하면 기본적인 것들이므로 폰갭(PhoneGap)보다는 활용이 좀 불편하다는 점은 분명히 존재합니다.
그리고 iOS나 Android 구분없이 대부분의 하이브리드 앱의 기능을 개발해 낼 수 잇다는 점입니다.
만약 운영체제에 따른 개발을 각각 해야한다면 비용과 시간 두가지 모두 많이 들겠죠.
코르도바는 이러한 문제를 해결해 준답니다.
```
